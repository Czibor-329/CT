我们研究一个具有**驻留时间约束的柔性作业车间调度问题（FJSP）**，系统由**组合设备或级联设备**构成。

在组合设备中，一台设备包含 **6 个并行工艺腔体（PM）和 1 个机械手（TM）**，机械手负责在腔体之间搬运晶圆。

在级联设备中，系统分为 **两个处理段**：

- 前段设备：`PM1–PM6`，由机械手 `TM3` 管理
- 后段设备：`PM7–PM10`，由机械手 `TM2` 管理

两个处理段通过 **两个缓冲腔体（LLC：入口 Buffer，LLD：出口 Buffer）**连接。
Buffer 可以存放晶圆，并允许晶圆在其中停留。

调度策略需要在满足**加工顺序、资源容量约束以及驻留时间限制**的条件下，完成晶圆加工并优化整体性能（如 makespan）。

理想策略应能够自动学习以下行为：

1. **下游瓶颈感知**：
   当后续工序加工时间较长或资源较少时，前序工序应**主动延迟释放晶圆**，避免在 Buffer 或下游设备产生拥堵。
2. **并行能力不对称控制**：
   当**前段并行机器数量多而后段并行机器数量少**时，策略应控制前段加工节奏，以避免系统过早产生过多在制品。
3. **Buffer 调度利用**：
   LLC 和 LLD Buffer 既可以缓冲晶圆，也可能成为拥堵点，策略需要根据上下游负载情况合理利用 Buffer。

目标是训练一个具有**泛化能力的调度策略**，能够适应以下变化：

- 工序数量变化（例如 2–4 道工序）
- 各工序可用并行机器数量变化
- 加工时间参数变化

策略应能够在不同设备结构和加工参数下，自动学习合理的**节拍控制与延迟调度策略**。怎么基于深度学习和强化学习实现。目前对于固定的工序来说，可以基于每一台机器构造特征，但是如果因为工序发生变化，或者某一到操作并行机器数量发生变化，输入的特征向量维度发生改变，就需要重新训练模型，我想实现一个模型，能够泛化到小范围改动的工序中。
