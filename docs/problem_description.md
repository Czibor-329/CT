- When: 在讨论问题建模、任务边界和策略泛化目标时使用。

  - 调度必须满足加工顺序、资源容量约束和驻留时间限制。
  - 策略目标是优化 makespan 并提升对小范围结构变化的泛化能力。

## 问题定义
我们研究一个具有驻留时间约束的柔性作业车间调度问题（FJSP），系统由组合设备或级联设备构成。

在组合设备中，一台设备包含 6 个并行工艺腔体（PM）和 1 个机械手（TM），机械手负责在腔体之间搬运晶圆。

在级联设备中，系统分为两个处理段：

- 前段设备：`PM1–PM6`，由机械手 `TM3` 管理
- 后段设备：`PM7–PM10`，由机械手 `TM2` 管理

两个处理段通过两个缓冲腔体连接：

- `LLC`：入口 Buffer
- `LLD`：出口 Buffer

Buffer 可以存放晶圆，并允许晶圆在其中停留。

调度策略必须在满足加工顺序、资源容量约束以及驻留时间限制的条件下，完成晶圆加工并优化整体性能，例如 makespan。

## 目标行为
理想策略应自动学习以下行为：

1. 下游瓶颈感知：当后续工序加工时间较长或资源较少时，前序工序必须延迟释放晶圆，避免在 Buffer 或下游设备产生拥堵。
2. 并行能力不对称控制：当前段并行机器数量多而后段并行机器数量少时，策略必须控制前段加工节奏，避免系统过早产生过多在制品。
3. Buffer 调度利用：`LLC` 和 `LLD` 既可以缓冲晶圆，也可能成为拥堵点，策略必须根据上下游负载情况合理利用 Buffer。

## 泛化目标
目标是训练一个具有泛化能力的调度策略，能够适应以下变化：

- 工序数量变化，例如 2 到 4 道工序
- 各工序可用并行机器数量变化
- 加工时间参数变化

策略必须在不同设备结构和加工参数下，自动学习合理的节拍控制与延迟调度策略。

## 建模要求
- 输入特征必须与设备结构保持一致。
- 当工序或并行机器数量变化时，特征设计必须保持可扩展。
- 规范文档优先描述行为和约束，不描述具体模型结构。
